### Get Fills

Retrieves fill records for a specific subaccount on the exchange.

This function queries the dYdX indexer API to fetch fill data (executed orders)
associated with a particular subaccount. The endpoint being called is `/v4/fills`.

A fill represents a trade that has been executed on the exchange.

#### Parameters

| Parameter           | Location | Type               | Description                                                           |
| ------------------- | -------- | ------------------ | --------------------------------------------------------------------- |
| `address`           | query    | [Address]          | The wallet address that owns the subaccount.                          |
| `subaccount_number` | query    | [SubaccountNumber] | The identifier for the specific subaccount within the wallet address. |

:::details[Optional]

| Parameter                     | Location | Type                      | Description                                                                                                        |
| ----------------------------- | -------- | ------------------------- | ------------------------------------------------------------------------------------------------------------------ |
| `ticker`                      | query    | [Ticker]                  | The market symbol to filter fills by (e.g., "BTC-USD"). If not provided, fills for all markets will be returned.   |
| `ticker_type`                 | query    | [MarketType]              | The type of market to filter by.                                                                                   |
| `limit`                       | query    | [u32]                     | Maximum number of asset positions to return in the response.                                                       |
| `created_before_or_at_height` | query    | [Height]                  | Restricts results to positions created at or before a specific blockchain height.                                  |
| `created_before_or_at`        | query    | [DateTime]                | Restricts results to positions created at or before a specific timestamp (ISO 8601 format).                        |
| `page`                        | query    | [u32]                     | The page number for paginated results.                                                                             |

:::

#### Response

A promise that resolves to fill data containing details such as order ID, market, side (buy/sell), size, price, execution time, and other fill-specific information.
